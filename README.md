### Commit Message Convention
> 출처 및 참고 : [[협업] 협업을 위한 git 커밋컨벤션 설정하기](https://overcome-the-limits.tistory.com/entry/%ED%98%91%EC%97%85-%ED%98%91%EC%97%85%EC%9D%84-%EC%9C%84%ED%95%9C-%EA%B8%B0%EB%B3%B8%EC%A0%81%EC%9D%B8-git-%EC%BB%A4%EB%B0%8B%EC%BB%A8%EB%B2%A4%EC%85%98-%EC%84%A4%EC%A0%95%ED%95%98%EA%B8%B0#%EB%A9%94%EC%8B%9C%EC%A7%80-%EA%B5%AC%EC%A1%B0)

* 타입, 제목, 본문, 꼬리말의 형태로 작성한다.
  ```text
  type: [#issue] Subject // 제목: 50자 제한, 명확하고 핵심적인 내용을 간략하게 요약하여 적는다 (이슈는 optional) 
  
  body // 본문 : 필요하다면 자세하게 작성 (optional)
  footer // 꼬리말 (optional)
  ```  
  
* 타입 종류
  * feat : 새로운 기능이 추가되었을 경우
  * refactor : 리펙토링 했을 경우 (기능의 업데이트도 refactor로 빠짐)
  * fix : 버그 수정
  * docs : 문서 관련 업데이트
  * test : 테스트를 위한 커밋
  * rename : 파일 이름만 변경되는 경우
  * comment : 주석을 추가했을 경우
